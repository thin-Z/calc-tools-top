# 点击统计优化设计规格

> 日期：2026-06-27 | 项目：calc-tools.top（工具箱里）
> 状态：设计已批准

## 一、概述

优化首页点击统计系统，新增趋势标识、使用次数展示、搜索热词统计三个功能，增强用户发现工具的能力和社交证明效应。

## 二、数据层设计

### 2.1 点击数据结构升级

**旧格式（需兼容迁移）：**
```javascript
// toolbox_clicks
{ "mortgage": 150, "bmi": 80 }
```

**新格式：**
```javascript
// toolbox_clicks
{
  "mortgage": { total: 150, daily: { "2026-06-27": 5, "2026-06-26": 8 } },
  "bmi": { total: 80, daily: { "2026-06-27": 3, "2026-06-25": 7 } }
}
```

**迁移策略：** 读取旧数据时检测，若值为数字（旧格式），自动升级为新格式。

### 2.2 搜索热词数据结构（新增）

```javascript
// toolbox_search_terms
{ "房贷": 15, "个税": 10, "利息": 8, "BMI": 6 }
```

**存储上限：** 保留前 30 个热词，超过时淘汰频率最低的。

### 2.3 函数改动

| 函数 | 改动 |
|------|------|
| `getClicks()` | 兼容旧格式（数字 → 新格式） |
| `saveClicks()` | 无变化 |
| `incrementClick(toolId)` | 增加 today 字段自增 |
| 新增 `recordSearchTerm(term)` | 搜索热词跟踪 |
| 新增 `getTrendingTools()` | 计算趋势标签 |
| 新增 `getHotSearchTerms(n)` | 获取 Top N 热词 |
| 新增 `getTodayDailyClicks()` | 获取当天各工具点击数 |

## 三、UI 设计

### 3.1 趋势标识（Feature B）

**位置：** 工具卡片右上角

**标签类型：**
| 标签 | 条件 | 样式 |
|------|------|------|
| `🔥 今日热门` | 当天点击次数 ≥ 该工具日均点击的 2 倍 | 红底白字 `.trend-hot` |
| `⬆ 上升中` | 近 3 天日均点击 > 前 3 天日均点击的 1.5 倍 | 橙底白字 `.trend-up` |

**降级规则：** 超过 7 天无点击活动的工具不显示任何趋势标签。

### 3.2 使用次数展示（Feature C）

**位置：** 工具卡片描述文字下方

**格式：** `已使用 XXX 次`

**数据来源：** 复用 `total` 字段

**显示阈值：** total ≥ 1 时显示，0 时不显示

### 3.3 搜索热词（Feature D）

**位置：** 搜索框下方

**格式：** `🔥 大家都在搜：房贷 | 个税 | BMI | 利息`

**交互：** 点击热词自动填入搜索框并触发搜索

**显示数量：** 最多 6 个热词

## 四、影响范围

| 文件 | 改动类型 |
|------|----------|
| `js/site.js` | 数据层 + 3 个新函数 + init 更新 |
| `css/site.css` | 新增 `.trend-badge` `.trend-hot` `.trend-up` `.usage-count` `.hot-search` 样式 |
| `index.html` | 搜索框下方添加 `.hot-search` 容器 + 卡片中添加 `.usage-count` 容器 |
| `en/index.html` | 同中文版 |

**不涉及的页面：** 各工具详情页、博客页

## 五、不做的功能（YAGNI）

- ❌ 不存储完整时间戳序列
- ❌ 不做数据看板/管理后台
- ❌ 不服务端存储
- ❌ 不跨设备同步
- ❌ 不添加排序切换（保持现有热度排序逻辑）
