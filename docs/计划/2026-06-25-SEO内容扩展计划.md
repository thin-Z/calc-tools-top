## Phase 1：现有页面优化（本周）

### 1.1 给每个工具页加 FAQ Schema

在每个工具页的 `<head>` 中追加 `FAQPage` JSON-LD，针对该工具常见的 3-5 个问题。

**涉及文件**: `zh/bmi.html`, `zh/date-calc.html`, `zh/housing-fund.html`, `zh/mortgage.html`, `zh/tax2026.html`（及英文对应页）

**示例（BMI）**:
```json
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [
    {
      "@type": "Question",
      "name": "BMI 多少算正常？",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "BMI 在 18.5-24.9 之间为正常范围，25-29.9 为超重，30 以上为肥胖。"
      }
    },
    ...
  ]
}
```

### 1.2 每个工具页底部加内容区

在 `</main>` 前追加：
- 工具简介段落（150-200 字）
- 使用说明
- 常见问题（纯文本，与 FAQ Schema 对应）

### 1.3 加 BreadcrumbList Schema

每个工具页追加面包屑导航结构化数据。

**示例**:
```json
{
  "@context": "https://schema.org",
  "@type": "BreadcrumbList",
  "itemListElement": [
    { "@type": "ListItem", "position": 1, "name": "在线计算工具", "item": "https://calc-tools.top/" },
    { "@type": "ListItem", "position": 2, "name": "房贷计算器", "item": "https://calc-tools.top/zh/mortgage" }
  ]
}
```

### 1.4 工具间交叉链接

每个工具页底部加"相关工具推荐"区域，链接到 2-3 个相关工具。

**示例（房贷页面）**:
```html
<div class="related-tools">
  <h3>相关工具</h3>
  <ul>
    <li><a href="/zh/housing-fund.html">公积金贷款计算器</a></li>
    <li><a href="/zh/tax2026.html">个税计算器</a></li>
  </ul>
</div>
```

## Phase 4：规模化和变现（1 个月+）

### 4.1 工具继续扩充到 30+

| 类别 | 工具举例 |
|------|---------|
| 财务 | 复利计算器、定投计算器、理财收益计算 |
| 健康 | 卡路里计算、基础代谢率 BMR、运动消耗 |
| 生活 | 房贷提前还款、车贷计算、装修预算 |
| 教育 | 成绩计算、GPA 计算、学习时间规划 |

### 4.2 内容矩阵自动化

- 建立模板化内容生产流程（AI 辅助）
- 每工具保持 2-3 篇文章的更新频率
- 批量生成 → 人工审核 → 上线

### 4.3 流量监测与优化

- Google Search Console 每周检查：
  - 收录率（目标 95%+）
  - 平均排名（目标 Top 10）
  - CTR 优化（优化 title + description）
- Google Analytics 观察：
  - 跳出率
  - 停留时长
  - 工具使用率

### 4.4 变现准备

- 流量稳定后申请 AdSense
- 考虑内页广告位优化
- 观察是否需要接入其他联盟

## 建议执行顺序

先从 **Phase 1** 开始（本周完成），因为：
1. 不改动现有结构，风险最低
2. 立即提升 SEO 表现（FAQ、面包屑、内容区）
3. 为后续工具扩充打好模板基础

Phase 1 做完看到 Search Console 数据反馈后，再推进 Phase 2。
