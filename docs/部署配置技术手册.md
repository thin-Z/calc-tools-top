## 二、配置文件详解

### 2.1 `vercel.json`

> **注意**: 当前使用 Cloudflare Pages 部署，`vercel.json` 作为冗余配置保留，主要功能已在 `_redirects` 中实现。

```json
{
  "cleanUrls": true,           // 去掉 .html 后缀
  "trailingSlash": false,      // URL 末尾不加斜杠
  "headers": [
    {
      "source": "/(.*)",
      "headers": [
        { "key": "X-Content-Type-Options", "value": "nosniff" },
        { "key": "X-Frame-Options", "value": "DENY" },
        { "key": "Referrer-Policy", "value": "strict-origin-when-cross-origin" }
      ]
    }
  ],
  "redirects": [ ... ]  // 24 条 301 重定向（与 _redirects 重复）
}
```

**安全头说明**:
| Header | 作用 |
|--------|------|
| `X-Content-Type-Options: nosniff` | 防止 MIME 类型嗅探攻击 |
| `X-Frame-Options: DENY` | 禁止页面被嵌入 iframe（防点击劫持）|
| `Referrer-Policy: strict-origin-when-cross-origin` | 同源完整 referrer，跨源仅 origin |

### 2.2 `_redirects`（Cloudflare Pages 活配置）

**文件位置**: `calculator-site/_redirects`
**格式**: `<source> <destination> <status_code>`

当前规则分类：

**图片工具短路径重定向（10 条中文 + 10 条英文）：**
```
/zh/compress      /zh/image/compress  301
/zh/compress.html /zh/image/compress  301
...（convert/resize/base64/color-picker 同理）
```

**设计意图**: 允许用户同时通过 `/zh/compress` 和 `/zh/image/compress` 访问同一页面，兼容不同时期的链接策略。

> **与 vercel.json 的关系**: Cloudflare Pages 优先使用 `_redirects`，vercel.json 仅作为 Vercel 部署的备份。

### 2.3 `sitemap.xml`

**条目数**: 60 条 URL
**结构**: 首页 / 32 工具 × 2 语言 / 13 博客 × 2 语言

每条 URL 包含：
- `<loc>` — 页面地址
- `<xhtml:link rel="alternate">` — hreflang 中英文对应关系
- `<changefreq>` — 更新频率
- `<priority>` — 优先级（首页 0.9，工具页 0.8，博客 0.7）

**示例**:
```xml
<url>
  <loc>https://calc-tools.top/</loc>
  <xhtml:link rel="alternate" hreflang="zh-CN" href="https://calc-tools.top/"/>
  <xhtml:link rel="alternate" hreflang="en" href="https://calc-tools.top/en/"/>
  <xhtml:link rel="alternate" hreflang="x-default" href="https://calc-tools.top/"/>
  <changefreq>weekly</changefreq>
  <priority>0.9</priority>
</url>
```

### 2.4 `robots.txt`

```
User-agent: *
Allow: /
Sitemap: https://calc-tools.top/sitemap.xml
```

### 4.2 推送问题处理

国内 HTTPS push 经常 `Connection was reset`。

**方案 A：反复重试**
```bash
git push origin main
# 如果失败，等 5 秒再试
```

**方案 B：SSH 临时切换**
```bash
git remote set-url origin git@github.com:thin-Z/calc-tools-top.git
git push origin main
git remote set-url origin https://github.com/thin-Z/calc-tools-top.git
```

**方案 C：GitHub REST API（脚本化）**
参考 `update_docs.js` 和 `_push_temp.js` 的实现流程：
1. `POST /repos/:owner/:repo/git/blobs` — 创建文件 Blob
2. `POST /repos/:owner/:repo/git/trees` — 创建 Tree
3. `POST /repos/:owner/:repo/git/commits` — 创建 Commit
4. `PATCH /repos/:owner/:repo/git/refs/heads/main` — 更新 Ref
