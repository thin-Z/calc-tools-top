## 一、项目概述

为 calc-tools.top 增加 11 个热门文字处理工具，中英双语独立页面，纯前端实现，隐私优先（所有处理在浏览器本地完成）。

### 核心目标
- 首批上线 11 个文字工具，补充工具站品类
- 独立 URL 策略，SEO 友好
- 与现有 21 个计算器/图片工具架构一致
- 全部纯前端，零后端依赖

## 三、工具详细设计

### 3.1 字数统计 (word-counter.html)

**设计**: 字数统计 + 关键词密度 二合一页面

**字数统计模式**:
- 输入: 大 textarea，实时统计（oninput）
- 统计项: 总字符数、中文字数、英文单词数、段落数、句子数、行数
- 统计细分: 可见字符数、含空格字符数、中文字符数、英文字符数、数字数、标点数
- 额外: 阅读时间估算、说话时间估算
- 可视化: 字符分布占比（中文/英文/数字/标点）

**关键词密度模式**:
- 输入: 同一 textarea，模式切换
- 统计项: 词频 Top 20、关键词密度百分比
- 可视化: 排序列表（词 + 次数 + 占比%）
- 兼容: 中文按字/词分析，英文按单词分析

### 3.2 大小写转换 (case-converter.html)

**输入**: textarea + 8 种模式单选按钮/芯片
**模式**:
- 大写 (UPPERCASE)
- 小写 (lowercase)
- 首字母大写 (Title Case)
- 句子大小写 (Sentence case)
- 驼峰式 (camelCase)
- 蛇形式 (snake_case)
- 烤肉字母 (kebab-case)
- 交替大小写 (aLtErNaTiNg)

**输出**: 转换后文本 + 一键复制按钮
**特点**: 点击模式即刻转换显示

### 3.3 JSON 格式化 (json-formatter.html)

**输入**: textarea + 格式化/压缩 双模式
**校验**: JSON 语法校验，错误提示（行号+描述）
**输出**: 格式化（缩进 2 空格）/ 压缩后的 JSON
**额外**: 一键复制、字符数统计、错误时保留原文本

### 3.4 Base64 编解码 (base64.html)

**输入**: textarea + 编码/解码 切换
**输出**: 编码或解码后的文本
**特点**: UTF-8 安全（TextDecoder + Uint8Array，解决中文乱码），一键复制

### 3.5 URL 编解码 (url-encode.html)

**输入**: textarea + 编码/解码 切换
**输出**: 编码或解码后的 URL
**特点**: UTF-8 中文安全，一键复制

### 3.6 文本清理器 (text-cleaner.html)

**输入**: textarea + 复选框选择操作
**操作**（可多选组合）:
- 去除多余空格（保留单空格）
- 去除所有换行
- 去除空行
- 去除首尾空白
- 去除标点符号

**输出**: 清理后文本，一键复制

### 3.7 去除 HTML 标签 (html-stripper.html)

**输入**: textarea
**操作**: 剥离 HTML 标签，保留纯文本内容
**可选**: 可选择保留 `<br>`、`<p>`、`<b>` 等指定标签
**输出**: 纯文本，一键复制

### 3.8 文本对比 (text-diff.html)

**输入**: 左右两个 textarea
**操作**: 点击"对比"按钮
**算法**: 基于 LCS（最长公共子序列）的 diff 算法
**输出**: 左右并排差异高亮
- 绿色背景 + 下划线: 新增内容
- 红色背景 + 删除线: 删除内容
- 白色背景: 未变内容

### 3.9 UUID / 密码生成器 (generator.html)

**UUID 模式**:
- 生成 UUID v4
- 批量生成（1-10 个，下拉选择）
- 每个 UUID 独立复制按钮
- 格式: 标准 8-4-4-4-12 格式

**密码模式**:
- 长度滑块（8-64 位）
- 可选字符: 大写字母、小写字母、数字、特殊符号
- 密码强度指示条（弱/中/强/很强）
- 批量生成（1-10 个）
- 一键复制

### 3.10 阅读时间计算 (reading-time.html)

**输入**: textarea，实时统计
**统计**:
- 阅读时间估算（中文 300 字/分钟，英文 200 词/分钟）
- 说话时间估算（慢速/正常/快速）
- 总字数/总字符数
- 阅读难度等级（小学/初中/高中/大学/专业）

**交互**: 阅读速度滑块自定义调整

## 六、技术要点

### 6.1 隐私优先
- 所有处理逻辑在浏览器本地完成（Canvas/纯 JS）
- 不上传任何文本到服务器

### 6.2 编码安全
- 所有文件 UTF-8 without BOM
- Base64 编解码使用 `TextDecoder` + `Uint8Array`（避免中文乱码）

### 6.3 diff 算法
- 使用 LCS（最长公共子序列）算法实现文本对比
- 行级对比，着色标记增删差异

### 6.4 密码强度评估
- 弱: < 8 位或只有一种字符
- 中: 8-11 位且两种字符
- 强: 12-15 位且三种字符
- 很强: 16+ 位且四种字符

## 七、实现顺序建议

1. 字数统计 + 关键词密度（最热门，做二合一展示）
2. 大小写转换（逻辑简单，快速出活）
3. JSON 格式化（开发者常用，校验逻辑标准）
4. Base64 编解码（与现有图片 Base64 呼应）
5. URL 编解码（与 Base64 结构相似，可快速复制）
6. 文本清理器（多操作组合，需设计 UI）
7. HTML 标签剥离（正则处理，逻辑清晰）
8. 文本对比（diff 算法需单独实现）
9. UUID/密码生成器（随机算法，UI 交互略复杂）
10. 阅读时间计算（逻辑最简，收尾）
11. 首页整合（所有工具完成后统一注册）
