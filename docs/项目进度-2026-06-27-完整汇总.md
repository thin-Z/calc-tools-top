
## 2026-06-27（第十六次）— 修复线上首页 CSS 链接断裂
### 问题
线上首页 index.html 的 <link rel="stylesheet" href="css/site.css 缺失闭合引号，浏览器将整行吞入 href 属性：
css/site.css%20%20%20%20%3Clink%20rel=

导致所有 3 个 CSS 文件无法加载，全站样式彻底丢失。

### 修复
- 补全 site.css 的版本参数和闭合引号 css/site.css?v=2"
- 添加 text-tools.css 引用
- 移除 BOM 头
- 通过 GitHub REST API 推送至 main 分支（commit 7a35af5）
- Vercel 自动部署生效

### 教训
部署前应验证 HTML 标签完整性，避免 <link> 标签断裂导致 CSS 全站失效。

## 踩坑要点

### PowerShell 编码陷阱
- Set-Content / Out-File 默认 ANSI 编码 → **UTF-8 字符静默损坏**
- 正确做法：[System.IO.File]::WriteAllBytes() + [Text.Encoding]::UTF8.GetBytes()

### GitHub API 推送
- `git push` 不可用时，通过 GitHub REST API 操作：git/trees → git/blobs → git/commits → git/refs
- API base64 解码需用 TextDecoder 而非 atob()（不处理 UTF-8 多字节）
- raw.githubusercontent.com 国内超时 → 改用 api.github.com/git/blobs
- "Update is not a fast forward" — 必须先获取最新 ref SHA 作为 parent

### JS 部署检查
- Template literal 括号数必须平衡（383 open / 382 close 就会全站瘫痪）
- 部署前验证 JS 解析 + 浏览器打开首页确认
